# Зоны ответственности

Кто владеет какими фактами и что агент может править автономно.

## Владение разделами

| Раздел | Владелец (роль) | Канонический для фактов о |
|---|---|---|
| product/ | владелец продукта / тимлид | целях, приоритетах, метриках продукта |
| domain/ | тимлид + аналитик | терминах, сущностях, бизнес-правилах |
| architecture/ | тимлид / архитектор | слоях, паттернах, стеке, стиле, безопасности |
| ops/ | DevOps / дежурный | окружениях, конфигурации, релизах, инцидентах |
| decisions/ | команда (решение фиксирует тимлид) | принятых архитектурных решениях и их причинах |
| workflows/ | тимлид | процессах работы над задачами |
| current/ | агент + разработчики | текущем фокусе и статусе |
| _meta/ | тимлид | правилах самой базы |

## Автономия агента

| Действие | Разрешение |
|---|---|
| Обновлять `current/*` по итогам сессии | Автономно |
| Проставлять `verified_commit` после сверки с кодом | Автономно (сверка — по запросу) |
| Править ошибочные факты в stable-разделах | В том же PR, что и код, с явным упоминанием в описании PR |
| Создавать черновики ADR (`decision_status: proposed`) | Автономно как черновик; статус `accepted` ставит только человек |
| Менять `product/*`, `_meta/*`, `workflows/*` | Только по прямой просьбе человека |
| Переводить документы в `archived`, удалять файлы | Только по прямой просьбе человека |
| Разрешать противоречия между документами | Не молча: фиксация в progress.md + сообщить (см. [lifecycle.md](lifecycle.md)) |

## Ревью

- Апрув владельца обязателен только для `_meta/`, `product/` и смены статусов
  в `decisions/`. Правки `domain/`, `architecture/`, `ops/`, `workflows/`
  проходят обычное код-ревью любым членом команды; владелец раздела смотрит их
  на спринтовой сверке.
- CI-валидатор обязателен к прохождению (см. `.github/workflows/validate-memory-bank.yml`).
