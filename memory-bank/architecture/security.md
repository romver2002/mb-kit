## Валидация входных данных на границах

- Всё внешнее (HTTP, очереди, файлы, CLI) валидируется на границе presentation
  до входа в application: схемы DTO — типы, диапазоны, форматы, размеры.
- Инварианты домена дополнительно защищаются в domain: сущность нельзя создать
  в невалидном состоянии. Валидация на границе этого не отменяет.
- Пользовательский ввод не конкатенируется в SQL / shell-команды / пути файлов —
  только параметризованные запросы и allowlist-подход.
- Ошибки наружу — без внутренних деталей (стектрейсы, SQL, пути): код ошибки
  и безопасное сообщение; подробности — в логи.
