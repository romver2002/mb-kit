## Правила по окружениям

### Люди

- **local/dev** — можно всё: эксперименты, ломающие миграции, очистка данных.
- **stage** — только через CI/CD и миграции; ручные правки данных — с уведомлением
  команды (stage должен оставаться репетицией prod).
- **prod** — только через процесс релиза ([releases.md](releases.md)) или
  hotfix ([../workflows/hotfix.md](../workflows/hotfix.md)). Ручной доступ —
  read-only; любое исключение фиксируется. TODO(template): где фиксируется
  (канал/тикет).

### Агенту (Claude Code / Codex) — жёстче

| Действие | local | dev | stage | prod |
|---|---|---|---|---|
| Запуск приложения и тестов | да | да | нет | нет |
| Чтение логов/метрик | да | да | да | только по прямой просьбе |
| Изменение данных, миграции | да | только через CI | нет | **никогда** |
| Деплой | — | только через CI | нет | **никогда** |

- Агент **не подключается к stage/prod напрямую** (БД, SSH, консоли облака) —
  даже для «просто посмотреть», если человек явно не попросил.
- Команды с прод-учётками, прод-DSN или `--env prod` агент не выполняет,
  а показывает пользователю для ручного запуска.
- Секреты любых окружений — см. запреты в
  [../architecture/security.md](../architecture/security.md).
