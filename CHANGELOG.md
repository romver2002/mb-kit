# Changelog шаблона базы знаний

Версии самого шаблона (не документов — у них своё версионирование во frontmatter).

## 1.2.0 — 2026-07-05

Исправления по итогам состязательного ревью Фазы 0 (11 подтверждённых дефектов).

- `bump_frontmatter.py`: сверка по merge-base (было — по вершине base: ложные
  срабатывания на разошедшихся ветках); кириллические имена файлов больше не
  выпадают (`core.quotePath=false`); недостижимая база даёт код 2, а не «чисто»;
  переименование+правка ловится (`--name-status -M -z`, старый путь).
- `mb_lib.py`: чтение/запись с нормализацией в LF (`load`/`save`); устранён
  round-trip LF→CRLF и висячий `\r` в set_scalar. Добавлен `.gitattributes` (eol=lf).
- `mb_log.py`: frontmatter `updated` ставится сегодняшней датой, `--date` датирует
  только запись (было — откатывал `updated`, вызывая ложное MB050).
- Хуки `check_session_close.py`, `kb_log_read.py`: stdin читается как UTF-8
  (на русской Windows кириллические пути искажались cp1251).
- `.claude/settings.json`: пути хуков якорены к `$CLAUDE_PROJECT_DIR`.
- Валидатор MB015: люфт в один день (локальная дата разработчика vs UTC в CI).
- CI: `scripts/**` в триггерах (правки инструментов не запускали тесты);
  детерминированная дедупликация еженедельного issue.
- `git-hooks/pre-commit` фиксируется исполняемым (`--chmod=+x`).
- Правило MB071 (INFO): счётчик оставшихся примеров-заглушек «замените своим» —
  сигнал выжимки для downstream-проектов.
- Согласованы документы: подвалы `current/` без ручного bump; лимит длины 80/160
  вместо 60; оговорка про исключение `current/` из CI-рубежа.

## 1.1.0 — 2026-07-05

Фаза 0 плана развития: запись почти бесплатна, использование измеримо.

- `scripts/bump_frontmatter.py` + `scripts/git-hooks/pre-commit`: `version`/`updated`
  проставляет автоматика; CI-рубеж не пропускает правку тела без bump.
  Ручная механика версий удалена из инструкций агентов.
- `scripts/mb_log.py`: запись в `current/` одной командой
  (done/next/question/focus/discrepancy/problem/debt), с авто-bump frontmatter
  и предупреждением о превышении бюджета длины.
- Stop-hook (`scripts/check_session_close.py`): сессия Claude Code, менявшая код,
  не закрывается без датированной записи в `current/`.
- Локальная телеметрия чтения (`scripts/kb_log_read.py`, `kb_usage_report.py`):
  доля сессий, читающих базу; соблюдение протокола; мёртвые файлы. Лог гитигнорится.
- «Критерии сворачивания» в `_meta/lifecycle.md` — записанные заранее пороги провала.
- Еженедельный CI-отчёт: дедупликация (комментарий в существующий issue вместо нового).
- `memory-bank/_meta/policy.md`: политика прав агентов (наивысший приоритет),
  push/PR — `never`, продублировано в `.claude/settings.json` permissions.deny;
  валидатор проверяет флаги (MB080/MB081).
- Валидатор переписан (v2.0.0): реестр правил MB001–MB081, типизированная модель,
  один проход чтения, `--json`/`--strict`/`--ignore`/`--list-rules`, юнит-тесты в CI.
